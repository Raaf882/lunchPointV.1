# نادي نقطة انطلاقة — الموقع الكامل

موقع نادي "نقطة انطلاقة": صفحة رئيسية كاملة (من نحن، الأنشطة، تواصل معنا)، استبيان تفاعلي من 16 سؤالاً لاكتشاف المجال الريادي الأقرب لشخصية الزائر، دعم كامل للعربية (RTL) والإنجليزية (LTR)، ولوحة تحكم إدارية كاملة (CMS) لإدارة كل شيء دون لمس الكود.

> **التصميم:** مستورد فعليًا من حزمة handoff الصادرة عن Claude Design (`موقع نقطة انطلاقة.dc.html`) وأُعيد بناؤه بالكامل بـ HTML/CSS/JS عادي.
>
> **الخطوط:** التصميم يستخدم خطين غير متاحين كملفات فعلية بعد: **Ouhod** و**ArahamahHoms**، معرّفان في `styles.css` عبر `@font-face` يشيران إلى `frontend/assets/fonts/Ouhod.*` و`Arahamah.*`. حتى تتوفر، يظهر الموقع بخط **Tajawal** (نفس بديل التصميم الأصلي) مع **Poppins** للتفاصيل اللاتينية. أضف الملفات لاحقًا بنفس الأسماء وستُلتقط تلقائيًا دون تعديل الكود.
>
> **قناة واتساب:** الرابط الافتراضي `whatsapp.com/channel/PLACEHOLDER` — حدّثه من لوحة التحكم (إدارة المحتوى ← روابط التواصل الاجتماعي).

## البنية

```
نقطة-انطلاقة/
├── frontend/
│   ├── index.html       # الموقع العام (كل الأقسام + الاستبيان)
│   ├── login.html         # شاشة تسجيل دخول الأدمن
│   ├── admin.html        # لوحة التحكم الكاملة (محمية بجلسة دخول حقيقية)
│   ├── styles.css
│   ├── script.js
│   ├── translations.js   # نصوص عربي/إنجليزي، الأسئلة الـ16، ألوان المجالات
│   └── assets/            # شعارات النادي، fonts/، uploads/ (صور مرفوعة من لوحة التحكم)
├── backend/
│   ├── server.js          # كل نقاط الـ API
│   ├── db.js               # مخطط قاعدة البيانات + بذر القيم الافتراضية
│   ├── auth.js              # مصادقة الأدمن (اسم مستخدم + كلمة مرور)
│   ├── questions.js
│   └── package.json
├── database/               # ملف SQLite (يُنشأ تلقائيًا)
└── .env.example
```

## التشغيل المحلي

المتطلبات: [Node.js](https://nodejs.org) نسخة 18 أو أحدث.

```bash
cd backend
npm install
cp ../.env.example .env
```
افتح `backend/.env` وعدّل `ADMIN_USERNAME` و`ADMIN_PASSWORD` إلى بيانات دخولك الخاصة، ثم:
```bash
npm start
```
- الموقع: http://localhost:3000
- لوحة التحكم: http://localhost:3000/admin — ستُنقل تلقائيًا لشاشة تسجيل دخول مخصصة إن لم تكن قد سجّلت الدخول بعد؛ استخدم `ADMIN_USERNAME`/`ADMIN_PASSWORD` من `.env`. الجلسة تُحفظ في كوكي آمن لمدة 7 أيام، ويوجد زر "تسجيل الخروج" داخل اللوحة.

قاعدة البيانات (`database/data.sqlite`) تُنشأ تلقائيًا عند أول تشغيل، وتُبذر بمحتوى افتراضي (نص "من نحن"، والأنشطة الست) يطابق التصميم الأصلي — عدّله من لوحة التحكم متى شئت.

## لوحة التحكم — الميزات

### 1) نتائج اكتشف شغفك
جدول بكل من أكمل الاستبيان (بحث بالاسم/المدينة، فرز حسب التاريخ/الاسم/المجال)، مع إمكانية فتح أي سجل لعرض إجاباته الـ16 كاملة ونسب كل مجال، وزر تصدير CSV.

### 2) رسائل تواصل معنا
كل الرسائل الواردة من نموذج "تواصل معنا" العام في الموقع (بحث/فرز)، مع زر لتمييز الرسالة كـ"تمت الإجابة عليها"، وتصدير CSV.

### 3) إدارة المحتوى
- **من نحن:** تعديل النص (عربي/إنجليزي) وصورة اختيارية للقسم.
- **بيانات التواصل:** البريد الإلكتروني ورقم الهاتف/الواتساب الظاهران في الموقع.
- **روابط التواصل الاجتماعي:** قناة واتساب، إنستغرام، تويتر/X، سناب شات، تيك توك، يوتيوب، تيليجرام — أي حقل تتركه فارغًا لا تظهر أيقونته في الموقع.
- **الأنشطة والفعاليات:** إضافة/تعديل/حذف نشاط (عنوان، وصف، وسم، تاريخ، صورة اختيارية بالعربي والإنجليزي)، مع حالة **متاح / قريباً / منتهي** تظهر كوسم على بطاقة النشاط في الموقع العام. حالة "متاح" فقط هي التي تُظهر زر "سجّل الآن" (انظر أدناه)؛ "قريباً" تعرض "التسجيل يفتح قريبًا"، و"منتهي" تعرض "انتهت الفعالية" — كلاهما نص غير قابل للنقر.

### 4) تسجيلات الفعاليات
عندما تكون حالة نشاط ما "متاح"، يظهر زر "سجّل الآن" على بطاقته في الموقع العام ويفتح نموذجًا (Modal) لتسجيل الزائر (الاسم، البريد، الجوال، ملاحظة اختيارية) في تلك الفعالية تحديدًا. لوحة التحكم تعرض كل التسجيلات في تبويب منفصل، مع فلترة حسب الفعالية/الاسم/التاريخ، وتصدير CSV يحترم الفلتر الحالي.

كل التعديلات تُحفظ في قاعدة البيانات وتنعكس فورًا على الموقع العام عند تحديث الصفحة (لا حاجة لإعادة تشغيل الخادم).

## نظام التنبيهات

عند كل إرسال ناجح لأي من النماذج الثلاثة (اكتشف شغفك، تواصل معنا، تسجيل فعالية):

- **داخل اللوحة:** أيقونة جرس أعلى اللوحة (بجانب زر تسجيل الخروج) بعداد للإشعارات غير المقروءة، يتحدّث تلقائيًا كل 20 ثانية. الضغط عليها يفتح قائمة بآخر 50 إشعارًا (نوع النموذج، اسم الزائر، الوقت)؛ الضغط على أي إشعار ينقلك مباشرة للتبويب والسجل المرتبطين به (تفاصيل الاستبيان الكاملة، أو الرسالة/التسجيل مع تظليل مؤقت). زر ✓ بجانب كل إشعار غير مقروء لتمييزه فرديًا، وزر "تعليم الكل كمقروء" أعلى القائمة.
- **بريد إلكتروني:** إذا أعددت متغيرات SMTP (انظر أدناه)، يصل بريد فوري إلى `ADMIN_NOTIFICATION_EMAIL` بنوع النموذج، اسم الزائر وبيانات تواصله، ورابط مباشر يفتح لوحة التحكم على نفس الإشعار تحديدًا (`/admin?notif=...`) — حتى لو لم تكن مسجّلًا دخولك بعد، سيأخذك الرابط لشاشة الدخول ثم يعيدك لنفس السجل تلقائيًا بعد الدخول.
- **بدون إعداد SMTP:** التنبيهات داخل اللوحة تعمل بشكل طبيعي دائمًا؛ إرسال البريد فقط يُتخطّى بصمت (مع تسجيل الحالة في console الخادم) ولا يؤثر إطلاقًا على حفظ النموذج.

### متغيرات البيئة المطلوبة لتفعيل البريد فعليًا
أضف هذه القيم الحقيقية في `backend/.env` (`.env.example` يحتوي أمثلة توضيحية فقط):
- `ADMIN_NOTIFICATION_EMAIL` — البريد الذي تريد استقبال التنبيهات عليه.
- `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS` — بيانات خادم SMTP فعلية (مثال Gmail: `smtp.gmail.com` منفذ `587`، مع "App Password" وليس كلمة مرور الحساب العادية).

بدون هذه الخمس متغيرات مجتمعة، تعمل التنبيهات داخل اللوحة فقط دون بريد — وهذا وضع طبيعي وآمن لا يحتاج أي إجراء إضافي إن لم ترغب بالبريد حاليًا.

## التحقق من صحة البيانات (Validation)

كل نماذج الموقع (اكتشف شغفك، تواصل معنا، تسجيل الفعاليات) تستخدم نفس قواعد التحقق في الواجهة **والخادم معًا** (الخادم يرفض أي طلب مباشر للـ API لا يحقق الشروط، حتى لو تجاوز تحقق الواجهة):
- **الاسم:** إلزامي في كل النماذج.
- **البريد الإلكتروني:** يجب أن يكون بصيغة صحيحة (نص@نطاق.نطاق) — إلزامي في تسجيل الفعاليات، واختياري بصيغة صحيحة إن أُدخل في البقية.
- **رقم الجوال:** أرقام فقط (مع قبول + في البداية)، بطول 9-15 رقمًا — إلزامي في تسجيل الفعاليات، واختياري بصيغة صحيحة إن أُدخل في البقية.
- رسائل الخطأ تظهر بجانب الحقل المعني مباشرة، بلغة الواجهة الحالية (عربي/إنجليزي).

## خطوات لاحقة عند الرغبة بالنشر العام

عند جاهزيتك للنشر (Render، Railway، أو VPS خاص):

1. **متغيرات البيئة:** أضف `ADMIN_USERNAME` و`ADMIN_PASSWORD` (و`PORT` إن احتجتها) كمتغيرات بيئة في لوحة تحكم المنصة بدلًا من `.env` المحلي. أضف أيضًا `NODE_ENV=production` كي تُرسَل كوكي الجلسة بعلامة Secure (تتطلب HTTPS، وهو متوفر تلقائيًا على Render/Railway).
2. **قاعدة البيانات:** SQLite مناسبة للاستخدام الحالي. عند الحاجة لاحقًا لتوسّع أكبر، الانتقال يقتصر على استبدال `backend/db.js`.
3. **الصور المرفوعة:** تُحفظ في `frontend/assets/uploads/` على القرص المحلي للخادم. على منصات مثل Render، فعّل "Persistent Disk" واربطه بمسار `frontend/assets/uploads/` (بالإضافة إلى `database/`) كي لا تفقد الصور وقاعدة البيانات عند كل نشر جديد.
4. **أمر التشغيل:** `npm install && npm start` من داخل `backend/` (حدّد `backend` كمجلد جذر الخدمة).
5. **HTTPS ونطاق مخصص:** متوفران تلقائيًا على Render/Railway.

## أوامر ونقاط API مفيدة

| الأمر / المسار | الوصف |
|---|---|
| `npm start` (من `backend/`) | تشغيل الخادم |
| `GET /api/admin/export.csv` | تصدير كل ردود الاستبيان CSV (يتطلب تسجيل الدخول) |
| `GET /api/admin/messages/export.csv` | تصدير رسائل التواصل CSV (يتطلب تسجيل الدخول) |
| `POST /api/events/:id/register` | تسجيل زائر في فعالية (عام، يتطلب أن تكون حالة الفعالية "متاح") |
| `GET /api/admin/event-registrations` | تسجيلات الفعاليات، يقبل `?eventId=&q=&dateFrom=&dateTo=` (يتطلب تسجيل الدخول) |
| `GET /api/admin/event-registrations/export.csv` | تصدير تسجيلات الفعاليات CSV بنفس فلاتر أعلاه (يتطلب تسجيل الدخول) |
| `GET /api/site-content` | محتوى الموقع الديناميكي (عام، بدون مصادقة) |
| `GET /api/admin/notifications` | آخر 50 إشعارًا + عدد غير المقروء (يتطلب تسجيل الدخول) |
| `PATCH /api/admin/notifications/:id/read` / `.../read-all` | تعليم إشعار واحد أو الكل كمقروء (يتطلب تسجيل الدخول) |
